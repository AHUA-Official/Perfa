# Perfa 本科毕业论文项目总览

## 论文基本信息

- 论文类型：本科毕业设计论文
- 学科方向：计算机科学与技术 / 软件工程 / 智能运维
- 写作语言：中文正文，包含英文摘要
- 交付形式：Word 文档为最终交付，Markdown 作为审稿与版本控制源文件
- 题目暂定：基于大语言模型与 MCP 的服务器性能测试平台设计与实现
- 项目参考材料：Perfa 项目源码、`codeknowledge/` 项目知识库、`uestc_thesis/` 学校论文要求与模板

## 学校硬性要求

- 正文字数不少于 20000 字，建议初稿正文目标为 23000 字左右，避免格式调整后低于要求。
- 中文摘要 300-500 字。
- 参考文献不少于 15 篇。
- 外文文献中文翻译不少于 1500 字。
- 最终论文推荐使用 Word 格式完成查重、格式检查和提交。
- 知网查重率、校内互查率均不高于 15%；推荐校优论文不高于 10%。

## 论文定位

本文围绕 Perfa 项目展开，研究对象是一套面向服务器性能测试场景的多组件平台。系统以自然语言交互为入口，通过 LangChain Agent 理解用户意图，并借助 MCP Server 将服务器管理、工具管理、压测执行、报告生成等能力封装为标准工具，最终由部署在被测节点上的 Node Agent 执行真实监控与性能测试任务。WebUI V2 提供对话、服务器、报告和监控等可视化功能。

论文不定位为纯算法创新论文，而定位为工程设计与系统实现论文。主要价值在于将大语言模型编排、MCP 工具协议、服务器性能测试工具链和可观测性链路结合起来，形成一套可执行、可扩展、可验证的智能化性能测试平台。

## 预期主要工作

1. 分析传统服务器性能测试流程中工具分散、操作门槛高、自动化程度不足、报告整理成本高等问题。
2. 设计 Perfa 的四层系统架构，包括交互层、智能编排层、能力封装层和节点执行层。
3. 实现 Node Agent、MCP Server、LangChain Agent 和 WebUI V2 四个核心子系统。
4. 构建覆盖服务器注册、工具安装、压测执行、结果查询、报告生成和监控展示的端到端流程。
5. 通过功能测试、接口测试和典型性能测试任务验证系统可用性。

## 写作质量要求

- 正文避免模板化表达，不使用机械过渡词和空洞强调句。
- 技术章节按输入、处理、输出、设计理由展开，避免模块清单式堆砌。
- 引言、相关技术和研究现状必须使用真实可追溯文献，不编造来源。
- 实验与测试章节只使用真实测试数据；若临时规划图表，必须明确标记为待替换。
- 每章完成后进行规范合规检查和质量检查。
