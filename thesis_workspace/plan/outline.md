# Perfa 本科毕业论文大纲

## 题目

基于大语言模型与 MCP 的服务器性能测试平台设计与实现

## 摘要

摘要在正文主体完成后撰写，控制在 300-500 字。摘要应覆盖研究背景、系统目标、技术路线、实现内容、测试验证和结论，不写引用，不放图表。

## 第一章 绪论

目标字数：3500 字左右。

本章说明服务器性能测试在系统运维、容量规划和故障定位中的作用，分析传统性能测试流程在工具管理、操作复杂度、结果整合和自动化方面的不足，引出自然语言驱动的性能测试平台需求。文献综述部分围绕服务器性能测试、可观测性、自动化运维、大语言模型智能体和 MCP 工具协议展开。章节末尾说明本文主要工作和论文组织结构。

## 第二章 相关技术与需求分析

目标字数：4000 字左右。

本章介绍 Perfa 涉及的核心技术基础，包括服务器性能测试工具体系、Prometheus 指标采集与可观测性、MCP 工具调用协议、LangChain Agent 编排机制、FastAPI/Flask 后端服务、Next.js 前端技术栈。需求分析部分从用户角色、功能需求和非功能需求展开，明确系统需要支持服务器管理、节点 Agent 部署、测试工具管理、压测任务执行、报告生成、监控展示和对话式操作。

## 第三章 系统总体设计

目标字数：4500 字左右。

本章从总体架构层面描述 Perfa 的设计方案。系统划分为 WebUI V2 交互层、LangChain Agent 智能编排层、MCP Server 能力封装层和 Node Agent 节点执行层。重点说明四层之间的调用关系、核心数据流、部署拓扑、模块边界、接口组织和数据存储方式。本章需要配套系统总体架构图、请求处理流程图、部署拓扑图和模块关系表。

## 第四章 系统详细设计与实现

目标字数：6500 字左右。

本章围绕源码实现展开，是全文主体。Node Agent 部分说明监控采集、工具管理、BenchmarkExecutor 和 HTTP API；MCP Server 部分说明服务器管理、Agent 生命周期、工具管理、Benchmark 和报告 Tool 的注册与调用；LangChain Agent 部分说明 OpenAI 兼容接口、流式返回、AgentOrchestrator、MCPToolAdapter 和工作流机制；WebUI V2 部分说明对话页、服务器页、报告页、监控页和前端代理层。章节需要突出输入到输出的技术链条，而不是简单罗列文件。

## 第五章 系统测试与结果分析

目标字数：4500 字左右。

本章设计测试环境、测试对象、测试方法和测试用例，验证系统功能完整性和端到端流程可用性。测试内容包括 Node Agent 健康检查、监控指标采集、工具安装与校验、压测任务执行、MCP Tool 调用、LangChain Agent 对话接口、WebUI 页面联调和报告生成。若有真实运行截图、接口返回和压测结果，应整理为表格和图。所有结果结论必须由真实测试材料支撑。

## 第六章 总结与展望

目标字数：1500 字左右。

本章总结本文完成的系统设计、核心实现和测试验证结果，客观分析当前系统在安全权限、异步任务调度、分布式节点管理、测试数据规模和报告智能化方面的不足，并提出后续优化方向。
